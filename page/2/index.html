深入理解JVM

字数统计: 1.2k 字 | 阅读时长 ≈ 4 分钟

你了解java内存模型吗?

程序计数器

  • 当前线程所执行的字节码行号指示器
  • 改变计数器的值来选取下一条需要执行的指令
  • 和线程是一对一的关系,线程私有
  • 不会发生内存泄露
  • 对Java方法计数,如果是Native方法测计数器值为Undefined

Java虚拟机栈

  • Java方法执行的内存模型
  • 每个方法执行在执行的过程中都会创建一个栈帧(局部变量表,操作数栈,动态链接,返回地址)
  • 每一个方法从调用直至执行完成的过程就对应着一个栈帧在虚拟机栈中入栈到出栈的过程
  • JVM中这个区域规定了2种异常:1.如果线程栈的深度大于虚拟机栈的深度:StackOverflowError2.如果虚拟栈动态扩展无法申请到足够内存:OutOfMemoryError

局部变量表和操作数栈

  • 局部变量表:包含方法执行过程中所有的变量
    其中64位的占用2个局部变量空间
  • 操作数栈:入栈,出栈,复制,交换,产生消费变量
  • 局部变量表和操作数栈的交互实例

本地方法栈

本地方法栈和Java虚拟机栈类似,只不过此栈是为虚拟机提供Native本地方法服务的.A native method is a Java method
whose implementation is provided by non-java code本地方法一般是用其它语言（C、C++ 或汇编语言等）编写的

Java堆

堆是JVM管理内存最大的一块,被所有线程锁共享,在JVM启动时创建,唯一目的是存放对象实例,堆还是GC管理的主要区域.

方法区

主要用于存储已被虚拟机加载的类信息(类镜像),常量,静态变量,即时编译器编译后的代码数据.
是各个线程所共享的内存区域.

元空间(MetaSpace)和永久代(PermGen)的区别

  • 元空间使用本地内存,永久代使用JVM内存
  • 在jdk7中,常量池从方法区移动到了堆区
  • String.intern()产生的结果不同

运行时的常量池

在jdk7以前运行时的常量池是方法区的一部分,常量池用于存放编译器生成的各种字面量和符号引用
这部分内容在类加载后进入方法区的运行时常量池中存放.特征是具备动态性除了在编译期(预置入常
量池内容)能产生常量,运行期也能将新的常量放入池内.

JVM三大性能调优参数含义

java -Xms128m -Xmx128m -Xss256k -jar xxx.jar

  • -xss :每个线程虚拟机栈的大小
  • -xms :堆的初始值
  • -xmx :堆能达到的最大值

String.intern()在jdk6和jdk6+的区别

  • jdk6:调用方法时,如果常量池先前已创建该字符串对象,则返回池中引用
    否则将字符串对象添加到常量池中,并返回常量池中的引用

  • jdk6+:调用方法时,如果常量池先前已创建该字符串对象,则返回池中引用,
    否则如果字符串对象已经在堆中,那么将堆中引用添加到字符串常量池,并且返回
    该引用(堆中的引用),若不存在则直接在常量池中创建该对象并返回其引用

堆和栈的区别

  • 联系:引用对象,数组时,栈里定义变量保存堆中目标的首地址
  • 空间大小,效率,碎片相关
  • 管理方式:栈自动释放,堆要GC
  • 分配方式:堆仅支持动态分配

Redis(Nosql内存数据库)

Redis是用C语言开发的一个开源的高性能键值对数据库

  • 数据类型包括:字符串类型 ●散列类型 ●列表类型 ●集合类型 ●有序集合类型

  • 内存持续和持久化
    Redis的所有数据存储在内存,内存的读取速度比硬盘高很多,所以性能更占优
    但程序退出后内存的数据会丢失,Redis提供持久化操作将内存数据异步存储到
    硬盘不影响继续提供服务

  • 多种功能
    多用于缓存和队列系统:
    Redis可以为每个键设置生存时间（Time To Live，TTL），
    生存时间到期后键会自动被删除这个功能可作为缓存系统使用
    Redis和Memcached优劣:
    在性能上Redis是 单线程模型，而Memcached支持多线程，所以在多核服务器上后者的性能更高
    如果需要用到高级的数据类型 或是持久化等功能，Redis是很好的替代品
    作为缓存系统，Redis还可以限定数据占用的最大内存空间，在数据达到空间限制后可以 按照一定的规则自动淘汰不需要的键
    Redis的列表类型键可以用来实现队列，并且支持阻塞式读取，可以很容易地 实现一个高性能的优先级队列。同时在更高层面上，
    Redis还支持“发布/订阅”的消息模式，可 以基于此构建聊天室等系统。

Redis持久化:
RDB是Redis默认采用的持久化操作(通过快照文件,将数据从硬盘载入到内存)
save 900 1 表示15分钟（900秒钟）内至少1个键被更改则进行快照
AOF(手动配置,操作一次写入一次数据)

函数依赖

  1. 记A->B 表示A函数决定B(B依赖于A)
  2. 对于 A->B，如果能找到 A 的真子集 ‘A’，使得’A’-> B，那么 A->B 就是部分函数依赖，否则就是完全函数依赖。
  3. 对于 A->B，B->C，则 A->C 是一个传递函数依赖。
  4. 如果 {A1，A2，… ，An} 是关系的一个或多个属性的集合，该集合函数决定了关系的其它所有属性并且是最小的，那么该集合就称为键码。

事务管理

  • 定义:事务处理用来维护数据库的完整性，保证成批的 SQL 语句要么全部执行，要么全部不执行
  • 事务四大特性：ACID
    • 原子性:操作要不全部成功,要不全部失败回滚
    • 一致性:事务执行前后都必须处于一致状态
    • 隔离性:并发事务互不干扰
    • 持久性；事务提交成功,对数据库的数据改变是永久的
