创作初衷

如何下载启动

后端

  1. 下载jdk8,redis,mysql,maven,git
  2. 下载Intelij idea
  3. git clone https://github.com/zljin/ms-app
  4. 进入db目录,将表创建到mysql中
  5. 修改application.yml的数据源配置文件
  6. 后端启动，访问http://localhost:8090/swagger-ui.html

前端

  1. https://m.imooc.com/wiki/uniapp-createframework 通过这个指引下载uni-app的基本环境，开发工具HBuilderX
  2. https://m.imooc.com/wiki/uniapp-directorystructure 通过这个指引下载uni-ui的模板
  3. git clone https://github.com/zljin/ms-ui-app
  4. 导入到HBuilderX运行，注意提前启动后端项目

如何使用

业务规则可以看下此课程的第一章，项目效果，知道如何项目如何操作即可。
https://www.imooc.com/learn/1079

和他的项目操作对比：
本项目在UI和功能上得到了加强，且必须登录后才能下单，登录后5分钟内失效否则继续登录即可

如何阅读源码

打开谷歌浏览器启动前端，点击按钮和操作时，可以看调用了那些后端接口，debug看细节即可，UserInterceptor为拦截器，可以看看如何拦截请求的，这是每个请求的入口。

总结

作为springboot项目开发，这套后端模板算是企业比较标准的写法，大家可以通过这个模板快速开发自己的应用，上家公司可能还没我这标准，汗。
麻雀虽小，五脏俱全。业务虽然简单，且这个秒杀的方案还有更多更好的设计。但胜在整套项目整体闭环，帮助同学入门是挺不错的素材。
最近在弄devops，也是对之前学习的复习了，借此机会，分享给大家。
