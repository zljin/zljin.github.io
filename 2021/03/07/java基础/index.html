    Java基本概念

标识符:自己所声明的类名,方法,变量统称标识符 规则：1.只能以”_”,”$”,character开头 2.区分大小写 3.无长度限制

关键字:java定义的具有特殊含义的单词

数据类型

  • 8大基本数据类型:long double(8) int float(4) short char(2) byte boolean(1)
  • 包装类型:将基本数据类型进行封装,形成引用数据类型,可自动装箱拆箱(常用于集合)
  • 引用类型:1.类类型 2.接口类型3.数组类型:数组长度固定不变,必须初始化

==java中的参数传递都是值传递(看成副本传递更好理解)==

java是按值传递,对于方法入参,若为基本数据类型,传入的参数为该参数的副本,副本值的改变与真正参数的值并无直接关系
若为引用数据类型,传入的参数为引用对象的地址值的副本,虽然地址值的副本,但还是指向目标的对象,所以能够改变.

  • Java中数据类型分为两大类：基本类型和引用类型。相应的，变量也分这两种类型：基本类型和引用类型。

==a.基本类型的变量保存原始值，即它代表的值就是数值本身；==

==b.引用类型的变量保存的值是引用值(“引用值”指向内存空间的地址,代表某一个对象的引用而不是对象本身)==

==c.包装类因为会自动装箱拆箱，所以可以和基本类型一样处理==

变量

  • 全局变量:定义在类中,方法体外,有默认值,自动初始化
  • 局部变量:定义在方法中,无默认值,使用前必须初始化
  • 类变量(static)

运算符:^异或 >>=右移 >>>=无符号右移

Object

  • 对于基本类型，== 判断两个值是否相等，基本类型没有 equals() 方法。
  • 对于引用类型，== 判断两个变量是否引用同一个对象，而 equals() 判断引用的对象的内容是否等价。如String.equals()比较的就是字符串内容是否相等。
  • equals()可重写,如果重写该方法就有自己的比较逻辑

String Pool

保存着所有字符串的字面量,在编译时期就确定.还可以使用intern方法在运行的过程中将字符串加入到String Pool中,并返回pool中的引用.

在 Java 7 之前，String Pool 被放在运行时常量池中，它属于永久代(方法区)。而在 Java 8，String Pool 被移到堆中

对象转换

面向对象

类:具有相同属性相同名称的抽象描述

对象:有状态,是该类某一瞬间的取值

属性:对象的特征 方法:对象的各种行为

方法重写与重载

重写:==返回值类型==和方法名相同且具备继承关系

重载:同一个类中,方法名相同,==参数列表不同==,==注意与返回类型无关==

权限标识符

  • public:任何一个类都可访问
  • private:只能在本类访问
  • default:只要同包即可访问(同包即可访问)
  • protected:同包||不同包的子父类可以访问(关注与父子类继承即可访问)

构造方法：创建对象并对全局进行初始化

oop三大特征

封装

将公共数据和复杂行为进行隐藏,并对外提供访问接口

继承
多态

同一种类型的事物具有多种不同的表现形式(父类引用指向子类对象)

规则:继承为前提,==成员方法==->编译看左边,运行看右边
==全局变量||静态方法==->编译看左,运行看左

static 关键字

  • 静态变量:称为类变量,属于类的变量,类的所有实例所共享,可通过类名直接访问,静态变量在内存中只存在一份.

  • 静态方法:类加载时就存在,不依赖任何实例,所以静态方法必须实现,不能有抽象方法

    总结:
    (1)随着类的加载而加载
    (2)该类所有对象所共享
    (3)优先级高
    (4)静态不能访问非静态(先人不知后人)

  • static代码块:

    • 用途:一次性对静态成员变量进行赋值
    • 特点:静态语句块在类初始化时只运行一次。

  • 静态内部类

    初始化顺序:
    a.静态变量 b.静态语句块 c.实例变量 d.普通语句块 e.构造函数

final 关键字

方法:子类不能重写,类:子类不能继承,数据：1.基本类型->数值不变2.引用类型-> 引用不变

集合

  • 数组:存放数据的容器,同一个数组中只能存放同一种类型的数据(元素),可以存储是引用类型或者基本数据类型的数据.
  • 集合:存放数据的容器,只能存储引用数据类型,在无泛型规定时可以添加任意的引用数据.
  • 集合的分类:单列集合(collection),双列集合(map),单列集合分为list和set
  • Iterator(java.util)是集合最顶层的接口,用于迭代访问集合中的元素
  • ListIterator(java.util) 专用于List的迭代接口，强化了父类方法
  • Iterable(java.lang)是集合的顶级接口之一,让该接口的子接口或者实现类具有可迭代性
    • Iterator iterator() 返回一组T类型元素进行迭代的迭代器,用于迭代遍历集合中的元素

Collection 单列集合接口

List

有序、可重复、有索引

  • ArrayList初始容量大小是10，增量是50%，在创建ArrayList对象的时候可以指定容量大小；Vector初始容量10；增量100%；(Vector:List的前生,但是后来被List收购)
  • ArrayList是线程不安全的，不同步的，但是效率高；Vector是线程安全的，同步的，但是效率低；

Set

无序、不可重复

  • HashSet-主要用于数据检索,通过Hash值排序

    • 线程不同步，效率高
    • ==如果在迭代遍历的过程中，要给Set集合中添加元素，则需要将Set集合转化成List集合；将List集合转成Set集合的时候一定要注意重复元素会丢失==
    • 保证元素不可重复的机制：只要两个对象调用==hashCode的值相等==，并且调用==equals方法的返回值为true==，则表示这两个对象相等，此时，只能有一个添加到Set集合中

  • TreeSet-红黑树,用于排序

    • 无序,默认内部是按照元素的字典顺序进行排序的.
    • ==在TreeSet中添加的元素所属的类需要实现Comparable接口==
    • 可以自定义排序规则:
      • 实现Comparable接口，重写compareTo方法；
      • 实现Comparator接口，相当于自定义了比较器，重写compare方法；

Comparable与Comparator

  • Comparable：强行对实现它的每个类的对象进行整体排序。称为==类的自然排序==实现此接口的对象列表（和数组）可通过Collections.sort和Arrays.sort进行自然排序，对象可以用作有序映射中的键或有序集合中的元素。
  • Comparator：强行对某个对象进行整体排序,称为==类的比较器==，优先级高于comparable接口

Map 双列集合接口

  • Map<==K==,V>集合中,==K为Set集合,所以Key类型必须一致且不同==
  • Properties:双列集合:<K,V>映射

异常

异常处理机制主要回答一下三个问题：

  1. what:异常类型回答了什么被抛出异常
  2. where:异常堆栈跟踪回答了在哪抛出
  3. why:异常信息回答了为什么被抛出

意义：大大降低编写和维护可靠程序的门槛

异常体系：

ERROR:程序无法处理的错误，编译器不检查 (JVM的责任)

Exception:程序可以处理的异常，捕获后可能修复

    RuntimeException：不可预知的，程序应该可能避免 (程序的责任)

    非RuntimeException：可预知的，编译器进行校验的异常(编译器的责任)

泛型提供了编译时类型安全检测机制,确保正确类型对象放入集合中。本质是将类型参数化

  1. <? extends T>表示该通配符所代表的类型是 T 类型的子类。
  2. <? super T>表示该通配符所代表的类型是 T 类型的父类

反射

为什么要使用反射？

  • 在编码阶段,要在配置文件中读取要使用的类，使用反射更加的灵活(如Spring在xml里创建一个bean对象)
  • 在开发时,程序需要依赖一个未知的类(如jdbc用mysql还是oracle),或者未开发完的接口实现类,可先获取其接口的结构进行编码

    相关概念

  1. ==运行状态中==，对任何一个类都可以知道这个类的所有属性和方法，对任意一个对象都可以任意调用属性和方法，这种==动态获取类信息和动态调用对象方法==称为Java的反射机制。
  2. 每个类都有一个class对象，包含了与类有关的信息，在编译新类时，会产生同名的.class文件，该文件内容保存着Class对象

IO

  1. 磁盘操作：File

  3. 字符操作：Reader 和 Writer

装饰者模式

  1. java的IO流采用装饰着模式,InputStream是抽象组件,FileInputStream是具体组件,提供了字节流的输入操作;
  2. BufferedInputStream属于抽象装饰者，强化了被装饰对象为FileInputStream提供缓存功能。

字节流与字符流操作

对象流(序列化)

提供了一种对象序列化的机制.用一个字节序列可以表示一个对象,该字节序列包含该 对象的数据,对象的类型和对象中存储的属性等信息.字节序列写出到文件之后,相当于文件中持久保存了一个对象的信息该字节序列还可以从文件中读取回来,重构对象,对它进行反序列化.(将对象进行冰冻解冻)

  • 使用要求
    • 对象必须实现 java.io.Serializable 接口
    • 该类的所有属性必须是可序列化的,若有个别属性不需要序列化,用transient修饰

其他增强流

  • 转换流InputStreamReader,OutputStreamReader
  • 序列流SequenceInputStream多个流一起处理
  • 内存流ByteArrayOutputStream

NIO

流与块

NIO面向快，IO面向流
NIO是非阻塞的，充分利用资源

通道与缓冲区

  1. channel:对原i/o中的流的模拟，但是双向的，可操作读&写
  2. buffer:发送和读取通道的所有数据首先要经过缓冲区。缓冲区是一个提供对数据的结构化访问和跟踪系统的读/写进程的数组

    通道常用类型

  • FileChannel：从文件中读写数据；
  • DatagramChannel：通过 UDP 读写网络中数据；
  • SocketChannel：通过 TCP 读写网络中数据；
  • ServerSocketChannel：可以监听新进来的 TCP 连接，对每一个新进来的连接都会创建一个 SocketChannel。

缓冲区状态变量

capacity:最大容量,position:当前已读的字节数,limit:还可以读写的字节数

过程：read->filp->write->clear 这些操作本质就是改变缓冲区的状态变量
