集合

Java中的集合体系理论

  • 数组:存放数据的容器,同一个数组中只能存放同一种类型的数据(元素),可以存储是引用类型或者基本数据类型的数据.
  • 集合:存放数据的容器,只能存储引用数据类型,在无泛型规定时可以添加任意的引用数据.
  • 集合的分类:单列集合(collection),双列集合(map),单列集合分为list和set
  • Iterator(java.util)是集合最顶层的接口,用于迭代访问集合中的元素
    • boolean hasNext()
    • E next()
    • void remove()
  • Iterable(java.lang)是集合的顶级接口之一,让该接口的子接口或者实现类具有可迭代性
    • Iterator iterator() 返回一组T类型元素进行迭代的迭代器,用于迭代遍历集合中的元素

Iterable 和 Iterator 的区别和意义是什么呢?

Collection(单列集合接口)

List(有序,可重复,有索引)

  • ArrayList初始容量大小是10，增量是50%，在创建ArrayList对象的时候可以指定容量大小；Vector初始容量10；增量100%；(Vector:List的前生,但是后来被List收购)

  • ArrayList是线程不安全的，不同步的，但是效率高；Vector是线程安全的，同步的，但是效率低；

  • ListIterator:List特有迭代对象(功能更加强大)

    • List遍历

    • 注意事项:

      • 在使用Iterator遍历的过程中，不要使用集合的方法来操作集合，而是改用Iterator中的方法来操作集合。(并发修改异常错误,两个操作操作一个集合会容易发生错误)
      • 避免在集合迭代遍历时进行集合中元素的操作——添加，移除

Set(无序,不可重复)

  • HashSet(主要用于数据检索,通过Hash值排序)

    • 线程不同步，效率高
    • 如果在迭代遍历的过程中，要给Set集合中添加元素，则需要将Set集合转化成List集合；将List集合转成Set集合的时候一定要注意重复元素会丢失
    • 保证元素不可重复的机制：只要两个对象调用hashCode的值相等，并且调用equals方法的返回值为true，则表示这两个对象相等，此时，只能有一个添加到Set集合中

  • TreeSet(红黑树,用于排序)

    • 无序,默认内部是按照元素的字典顺序进行排序的.
    • 在TreeSet中添加的元素所属的类需要实现Comparable接口
    • 可以自定义排序规则:
      • 实现Comparable接口，重写compareTo方法；
      • 实现Comparator接口，相当于自定义了比较器，重写compare方法；

Comparable与Comparator

  • Comparable：强行对实现它的每个类的对象进行整体排序。这种排序被称为类的自然排序，类的compareTo方法
    被称为它的自然比较方法。只能在类中实现compareTo()一次，不能经常修改类的代码实现自己想要的排序。实现
    此接口的对象列表（和数组）可以通过Collections.sort（和Arrays.sort）进行自动排序，对象可以用作有序映射中
    的键或有序集合中的元素，无需指定比较器。

  • Comparator强行对某个对象进行整体排序。可以将Comparator 传递给sort方法（如Collections.sort或
    Arrays.sort），从而允许在排序顺序上实现精确控制。还可以使用Comparator来控制某些数据结构（如有序set或
    有序映射）的顺序，或者为那些没有自然顺序的对象collection提供排序

  • comparator:优先级高于comparable接口

  • comparable 中的方法

    • int compareTo(T o) 比较此对象与指定对象的大小。

  • comparator中的方法compare(T o)

Map(双列集合接口)

  • Map<K,V>集合中,K为Set集合,所以Key类型必须一致且不同.(重要)
  • Properties:双列集合:<K,V>映射
    • setProperty(“属性名”,”属性值”)
    • getProperty(“属性名”)
  • Map的遍历方式

泛型(只在编译时有效)

  • 广泛的类型可以看成是一个变量;该变量可以接受任意引用类型的数据类型

  • 泛型类,泛型方法,泛型接口

  • 泛型边界:

    • 泛型上限:<? extends X >?所能代表的泛型最大只能是X类型

      泛型下限<? super X>?所能代表的泛型的最小类型只能是X类型

枚举(enum)

  • 也算一个类,但不能创建对象
  • 即使在枚举类中也不能new对象，枚举类的对象就是在枚举类中定义的有限个的对象
