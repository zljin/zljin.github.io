类的加载机制和反射

类加载器机制

  • 类加载器负责将.class文件加载到内存,并生成对应的java.lang.Class对象.
  • 类的加载机制—- Person p=new Person();
    • 编码
    • 编译 javac .java—>.class
    • 运行 java Person ————> Person.class
    • 类加载:双亲委托机制–向上委托,向下加载:
      • 1.Bootstrap ClassLoader，启动类加载器
        $JAVA_HOME/lib
      • 2.External ClassLoader，外部类加载器
        $JAVA_HOME/jre/lib/exts
      • 3.Application ClassLoader，应用程序类加载器
        $CLASSPATH
        • Person.class文件加载到内存中，JVM会对该字节码文件进行分析，然后产生Class类的对象，
          把该对象称之为Person类的镜像，Class类是用于描述字节码文件这一类事物的。
        • Person类中的main方法

反射

  • Java 反射机制在程序运行时，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用它的任意一个方法和属性。这种 动态的获取信息 以及 动态调用对象的方法 的功能称为 java 的反射机制
