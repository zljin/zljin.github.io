多线程

1.并发与并行

  • 并发:指两个或多个事物在同一个时间段内发生
  • 并行:指两个或多个事物在同一个时刻发生(同时发生)

2.线程与进程

  • 进程:进程是程序的一次执行过程，是系统运行程序的基本单位.
  • 线程:线程是进程中的一个执行单元，负责当前进程中程序的执行，一个进程中至少有一个线程.
    (简要来说,进程就是抢占cpu的资源,然后调用线程去利用这些资源)

3.线程的2中创建方式

  • 新建一个类 继承Thread类,重写run()方法
  • 新建一个类 实现Runnable接口重写run()方法

4.常用静态方法

5.常用匿名内部类实现多线程

JDK7中:
匿名内部类中,只能出现final修饰的局部变量

JDk 8
出现在匿名内部类中,局部变量可以不用显示的加上final修饰
但是系统会默认有final修饰符,所以其值不可以改变

6.守护线程(备胎?)

  • setDaemon(boolean), 设置一个线程为守护线程, 该线程不会单独执行, 当其他非守护线程都执行结束后, 自动退出【守护线程 也可以守护 main线程】

7.线程的同步问题

多个线程抢占一个资源,保证每次只有一个线程抢占一个资源,其他必须等待这个线程结束才能继续公平的抢(火车站抢票)

  • synchronized锁

    • 同步代码块

    • 同步方法

  • Lock锁(互斥锁- 条件变量)

    • Lock锁也称同步锁，方法如下：

      • public void lock() :加同步锁。
      • public void unlock() :释放同步锁。

8.线程间的通信,同步锁

  • 什么是线程间的通信
    • 多个线程在处理同一个资源，但是处理的动作（线程的任务）却不相同(生产者消费者问题)
  • 锁对象调用wait() ,notify() 和 notifyAll()使用要求:
    • wait()和notify()必须在同一锁对象使用
    • wait()和notify()是Object的对象方法
    • 此方法必须在同步方法或者同步方法块中实现
