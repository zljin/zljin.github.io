JavaEE基础

ServletContext:

  1. Context上下文,ServletContext代表整个应用
  2. 一个应用只有一个ServletContext对象(单例对象)
  3. 作用:域对象：在一定范围内,区间内（当前应用），使多个Servlet共享数据。
  4. 其他域对象:session,request,page
  5. 常用方法:
    • void setAttribute(String name,object value)
    • Object getAttribute(String name)
    • void rmoveAttribute(String name)

HTTP协议:超文本传输协议

  1. 应用层协议 无状态
  2. URL(Uniform Resource Locator 缩写为URL)统一资源定位符
    schema://host[:port#]/path/.../[;url-params][?query-string][#anchor]
  3. HTTP组成:(图片)

4. HTTP其他常用的头,行请自行查询

Request和Response

  • HttpServletResponse:

    • 响应编码

      response.setHeader("Content-Type","text/html;charset=utf-8");

    • 方法

      • getWrite(); 字符输出流
      • getOutputStream(); 字节输出流

  • HttpServletRequest:

    • 接受编码request.setCharacterEncoding(“utf-8”);

      Get请求:
      tomcat8接收参数时采用了utf-8

      tomcat7接收参数时采用了IOS-8859-1编码

  • Servlet 转发和重定向:

    1. 转发: 以前的request中存放的变量不会失效，就像把两个页面拼到了一起—地址栏不会变，客户端发送一次请求
      • request.getRequestDispatcher(“/index.jsp”).forward(request,response);
    2. forward 和 include的区别:
      forward原先执行的servlet终止,include原先的Servlet还可以继续响应
    3. 重定向: 以前的request中存放的变量全部失效，并进入一个新的request作用域。–地址栏会变，并发送2次请求，增加服务器负担
      • response.sendRedirect(“/index.jsp”);

Cookie和Session

  • 什么是会话?
    用户打开浏览器,访问服务器多种资源,然后关闭浏览器这个过程称为一次会话
  • 会话的作用?
    存储某个用户这个会话过程的数据,如购物车存储用户购买信息

    Cookie(存放在客户端的会话技术)

  • 如密码保存的cookie保存在客户端
  • 常用方法

Session(存放在服务器端的会话技术)

  • 在服务器为每一个用户开辟一个独享的HttpSession的对象在进行使用.
  • 常用方法

jsp(动态页面技术)

  • jsp本质就是servlet,访问index.jsp的时候会先编译成index.java再编译成index.class

  • jsp用于前端显示界面(重点是显示数据)，servlet:控制器(写逻辑)

  • 3个指令 page include taglib

  • 6个常用动作

  • 9大内置对象

    • 内置对象是指在JSP的<%=%> 和<% %>中可以直接使用的对象

重点 pageContext

  • 本身也是一个域对象：它可以操作其它三个域对象（request session application）的数据

  • pageContext获取其他8个隐式对象

  • 调用其他域的常用方法

  • findAttribute(String name) 从page request session application依次查找，找到了就取值，结束查找

四大域对象

  • PageContext : pageConext 存放的数据在当前页面有效。开发时使用较少。
  • ServletRequest: request 存放的数据在一次请求（转发）内有效。使用非常多。
  • HttpSession: session 存放的数据在一次会话中有效。使用的比较多。如：存放用户的登录信息，购物车功能。
  • ServletContext: application 存放的数据在整个应用范围内都有效。因为范围太大，应尽量少用。
