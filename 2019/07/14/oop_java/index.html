OOP基础:

基本概念:

类与对象
  • 类是对象的抽象，而对象是类的具体实例。类是抽象的，不占用内存，而对象是具体的，占用存储空间。
  • 类是用于创建对象的蓝图，它是一个定义包括在特定类型的对象中的方法和变量的软件模板。
方法重写与重载
  • 重写:返回值类型和方法名相同且具备继承关系
  • 重载:同一个类中,方法名相同,参数列表不同,注意与返回类型无关

三大特性

封装:定义:将公共数据和复杂行为进行隐藏,并对外提供访问接口
继承:

  • 规则:

    1. 默认情况,子类所有构造器都会调用父类的无参构造器
    2. 子比父好

  • 抽象类与接口:

    1. 抽象类
      • 抽象方法:没有方法体的方法,如果一个类包含抽象方法,那么这个类必须声明为抽象类
      • 普通类区别:抽象类虽有构造器但不能实例化,需要继承抽象类才能实例化其子类.
      • 子类必须实现所有的抽象方法
    2. 接口
      • 抽象类延伸
      • 方法默认被public abstract 修饰
      • 变量默认被public static final 修饰
      • 接口可实现多重继承
多态:
  • 定义:同一种类型的事物具有多种不同的表现形式(父类引用指向子类对象)
  • 规则:继承为前提,成员方法->编译看左边,运行看右边—-全局变量||静态方法->编译看左,运行看左

OOP常问:

static关键字
  1. 静态变量:称为类变量,属于类的变量,类的所有实例所共享,可通过类名直接访问,静态变量在内存中只存在一份.
  2. 静态方法:类加载时就存在,不依赖任何实例,所以静态方法必须实现,不能有抽象方法
  3. 总结:
    • 随着类的加载而加载
    • 该类所有对象所共享
    • 优先级高
    • 静态不能访问非静态(先人不知后人)
final关键字
  1. 方法:子类不能重写
  2. 类:不允许被继承
  3. 数据:对于基本类型->final使数值不变,对于引用类型->final使引用不变
String,StringBuffer,StringBuilder区别
  1. 可变性
  2. 线程安全 StringBuffer线程安全,内部使用synchronized 进行同步
  3. String Pool(字符串常量池)
Object类
  • equals():
  • hashcode():返回散列值,等价的两个对象散列值一定相同，但是散列值相同的两个对象不一定等价。
  • clone():深拷贝,浅拷贝
