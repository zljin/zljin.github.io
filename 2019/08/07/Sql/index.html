分组(具有相同的数据值的行放在同一组中)

  • 分组规定
    • GROUP BY 子句出现在 WHERE 子句之后，ORDER BY 子句之前,后面条件用having；
    • 除了汇总字段外，SELECT 语句中的每一字段都必须在 GROUP BY 子句中给出；
    • NULL 的行会单独分为一组
    • 大多数 SQL 实现不支持 GROUP BY 列具有可变长度的数据类型。
    • 分组后常用到的函数count(),sum(),avg(),min(),max()

连接(连接多个表)

  • 使用JOIN关键字,条件语句为ON
  • 内连接(inner join):等值连接,无条件语句下返回笛卡尔积
  • 自连接:内连接myself
  • 自然连接(natural join):
    与内连接的区别就是内连接提供连接的列，而自然连接自动连接所有同名列
  • 外连接(left/right/full outer join):
    两个表进行内连接,保留了没有关联的那些行(如左外连接就是保留左表没有关联的行)

组合查询 Union和Union all区别

使用 UNION 来组合两个查询，如果第一个查询返回 M 行，第二个查询返回 N 行，那么组合查询的结果一般为 M+N 行。
每个查询必须包含相同的列、表达式和聚集函数。
默认会去除相同行，如果需要保留相同行，使用 UNION ALL。
只能包含一个 ORDER BY 子句，并且必须位于语句的最后。

视图

  • 视图是虚拟的表，本身不包含数据，也就不能对其进行索引操作。
  • 优点:简化SQL操作,只使用表的部分,只给用户访问视图权限(安全)

存储过程

  • 对一系列SQL操作的批处理
  • 好处:代码封装(安全),代码复用,由于预先编译具有很高性能
  • 命令行中创建存储过程需要自定义分隔符
  • 包含 in、out 和 inout 三种参数,给变量赋值都需要用 select into 语句。
  • 每次只能给一个变量赋值，不支持集合的操作。
