事务管理

  • 定义:事务处理用来维护数据库的完整性，保证成批的 SQL 语句要么全部执行，要么全部不执行
  • 事务四大特性：ACID
    • 原子性:操作要不全部成功,要不全部失败回滚
    • 一致性:事务执行前后都必须处于一致状态
    • 隔离性:并发事务互不干扰
    • 持久性；事务提交成功,对数据库的数据改变是永久的

  • 基本术语

    • 事务（transaction）指一组 SQL 语句
    • 回退（rollback）指撤销指定 SQL 语句的过程
    • 提交（commit）指将未存储的 SQL 语句结果写入数据库表
    • 保留点（savepoint）指事务处理中设置的临时占位符placeholder可回退（与回退整个事务处理不同）。
