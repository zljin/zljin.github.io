基础语法

关键字：定义的一些具有特殊用途的单词

标识符：给程序中的变量、类、方法命名的符号
数据类型：分为基本数据类型、包装类型、引用类型
运算符：给程序进行数据的运算,赋值,比较的符号
流程控制：分支结构和循环结构

面向对象

类是一个具有相同特征和相同行为的抽象描述,对象是通过类创建具体实例.如人和人类的概念。人类有特征和行为,属性和方法对于其特征和行为

Object

顶层父类

equals() 比较对象是否相等
hashCode() 默认实现返回对象的内存地址

如果两个对象通过equals()比较是相等的，那么它们的hashCode()必须返回相同的值

==和equals()有那些区别?

  1. ==是运算符 一个是方法
  2. ==如果比较基本数据类型就是比较数值是否相等,比较对象则是比较地址是否相等
  3. equals() 用来比较对象是否相等。大部分子类都会重写，如String,Integer重写后都是比较值是否相等

java中的参数传递时值传递吗?
java中的参数传递都是值传递(看成副本传递更好理解)
a.基本类型的变量保存原始值，即它代表的值就是数值本身；
b.引用类型的变量保存的值是引用值(“引用值”指向内存空间的地址,代表某一个对象的引用而不是对象本身)
c.包装类因为会自动装箱拆箱，所以可以和基本类型一样处理

泛型

将类型参数化。定义在类,接口,方法时这个类型形参将在创建对象,调用方法时动态的指定

  1. 适用于多种数据类型执行相同的代码,提高代码复用性和扩张性
  2. 提供了编译时类型安全检查

注解

注解又称 Java 标注,可以通过反射获取标注内容(如被标注的方法,类),从而通过反射实现一些自定义功能

异常

异常就是何时何地报出什么类型的错误,用于程序的健壮性,容错性和系统的可维护

SPI

service provider interface: 允许应用程序在运行时动态发现和加载服务实现，本质上是一种基于接口的可插拔设计模式，实现了接口与实现的解耦

反射

在运行时能够动态的获取类的信息和动态调用对象的方法和属性的能力

集合

Iterable和Iterator

  1. Iterable接口是单列集合的顶层父类,实现这个接口的类具有可迭代性
  2. Iterable接口中的Iterator()方法用于迭代遍历集合元素
  3. 创建新的Iterable不用担心Iterator指针问题,互不影响
  4. 在迭代器遍历的时候不要去添加删除元素，会有并发问题同时操作

Map: 双列集合接口

HashMap 特点

1.数组+链表+红黑树
2.线程不安全、允许null键值、效率高
3.默认初始容量为16，默认负载因子为0.75 (当元素数量超过容量*负载因子时，进行扩容)

put()流程：
1.计算key的hash值(通过hashCode()高16位异或低16位，减少哈希冲突)
2.计算数组索引：(n-1) & hash
3.遍历链表/红黑树
找到相同key → 覆盖value
没找到 → 插入尾端
4.树化检查：链表长度≥8且数组长度≥64 → 转红黑树
5.扩容检查：size > threshold → 扩容2倍

IO

以应用程序为主体,往应用程序输入为输入流,从程序输出为输出流

Serializable：

  1. 序列化：将对象以字节序列的形式持久化到一个文件中。
  2. 反序列化：解析字节序列的文件并生成对象。(将对象进行冰冻和解冻,常用于流数据的传输等)
  3. 使用要求：该类实现Serializable 接口,transient修饰的属性不会序列化

BufferedReader：
装饰设计模式的增强类，不能单独使用要依赖被增强的对象
