LangChain4j+Cursor 快速开发AI聊天机器人

Created2025-10-16|Updated2025-10-18|project
|Post Views:

基础介绍

LangChain4J是Java的AI开发框架

步骤一：使用ChatModel用来与AI大模型交互 –> chat-model

我们用的AI大模型是阿里云百炼平台，需要获取api-key

步骤二：多模态，指的是能够同时处理不同类型数据的能力，如文本，图像，音频

步骤三：系统提示词，SystemMessage

系统Prompt相当于给AI设定人格和能力的边界，告诉AI “你是谁，你能做什么”

提示词优化技巧!! 可学!!

步骤四：AI Service,提供更高层抽象API

直接写接口和注解即可开发

步骤五：会话记忆

MessageWindowChatMemory 会话记忆，最多保存 N 条消息，多余的会自动淘汰

可通过messageId 映射不同的会话

步骤六：结构化输出

步骤七：RAG检索增强 —> EmbeddingModel

读取自己的知识库，将知识库进行切片和收集

转换成向量模型，并存储

自定义内容查询器

步骤八：工具调用，即AI借用外部工具，完成它做不到的事情，如获取天气预报的数据

步骤九：模型上下文协议 - MCP协议

此协议可以让AI轻松接入别人提供的服务实现更多功能，如查询地理位置、全网搜索等

你既可以直接接入网上第三方数据，也可以导入本地的其他MCP协议的服务

这些别人提供的第三方服务也是需要对应服务的api key的

https://mcp.so/zh

这里我们使用 智谱AI开放平台，获取其mcp的api-key,得到全网搜索的能力

https://bigmodel.cn/usercenter/proj-mgmt/apikeys

步骤十：护轨，即拦截器

步骤十一：AI服务化，SSE流式输入 —> streaming-chat-model

然后定义controller接口，自动注入SSE流式服务方法

让生成出来的答案响应流式返回

步骤十二，通过cusor快速编写前端对话系统

代码路径

https://github.com/zljin/ai-code-helper

由于application-local.yml存放一些密钥信息，则不提交了，仅供参考

如何启动？
springboot启动后端服务后

cd ai-code-helper-frontend
npm install
npm run dev
